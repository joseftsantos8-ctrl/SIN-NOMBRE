### Persistencia con localStorage (NUEVO — clave para usabilidad)
- Módulo [storage/persistencia.js](../storage/persistencia.js) con patrón **registry**: cada `data.js` se registra con un getter+setter; persistencia.js no importa nada de ellos, evitando dependencias circulares.
- Helpers `reemplazarArray()` y `reemplazarObjeto()` que mutan en sitio (mantienen la misma referencia que importaron otros módulos via `export let`).
- `guardarTodo()` con debounce de 200ms para evitar escrituras múltiples seguidas.
- `cargarTodo()` se ejecuta en `main.js` **antes** de cualquier render (línea 18).
- Hooks de auto-save en cada mutación: registrarMerma, registrarPedido, registrarAccion, setAsignacion, tasks.push (líder, ejecutar, inventario, negativos), perfil pic, crear usuario.
- Colecciones persistidas: `users`, `tasks`, `novedades`, `mermas`, `pedidos`, `levantamientos`, `auditoria`, `asignaciones`, `negativos`.
- **NO persistido**: `products` (20K items autogenerados, demasiado pesado). Si admin añade artículo, se pierde al recargar — esto es una limitación conocida.
- Storage key: `sirena_estado_v1`.
- **Reset disponible** desde tool card "Reset Sistema" del Admin, o desde consola: `SIRENA_RESET()`.
- Para forzar guardado manual desde consola: `SIRENA_GUARDAR()`.

### Mandato #1 — Evidencia fotográfica obligatoria (NUEVO)
- Módulos: [evidencias/data.js](../evidencias/data.js), [evidencias/camara.js](../evidencias/camara.js).

- Módulo [reportes/exportar.js](../reportes/exportar.js).
- 8 reportes: tareas, mermas, pedidos, auditoría, solicitudes, inventario materiales, consumos, evidencias.

### Mandato #9 — Escáner código de barras (NUEVO)
- Módulo [escaner/escaner.js](../escaner/escaner.js).
- Usa **BarcodeDetector API** nativa (Chrome/Edge móvil + desktop con cámara).
- Soporta: EAN-13, EAN-8, Code-128, Code-39, UPC-A, UPC-E.
- Detección cada 300ms mientras la cámara esté abierta.
- Fallback "Ingreso manual" si el navegador no soporta API.
- Busca el código en productos y materiales; muestra resultado.

### Mandato #10 — PWA básico (NUEVO)
- [manifest.webmanifest](../manifest.webmanifest): nombre, iconos (logo Grupo Ramos), theme color, modo standalone.
- [sw.js](../sw.js): Service Worker con cache-first para assets estáticos y network-first para módulos JS.
- Registro automático en `main.js`.
- Permite **instalar** la web como app desde Chrome/Edge ("Add to Home Screen" en móvil, instalable en desktop).
- Cachea assets para funcionar offline parcialmente.

### Notificaciones de tareas al login (NUEVO)
