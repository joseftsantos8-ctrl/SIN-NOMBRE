## ✅ Implementado

### 1. Modularización completa (Screaming Architecture)
- Migrado del `app.js` monolítico de 678 líneas a 20+ módulos ES.
- Cada dominio tiene su carpeta. `main.js` solo conecta listeners.
- Estado compartido vía `export let` mutable (`users`, `tasks`, `products`, `mermas`, `asignaciones`, `negativos`).
- Funciones llamadas desde HTML dinámico se exponen en `window.*` (`abrirModalEjecutar`, `abrirModalResultados`, `abrirModalPerfil`, `abrirModalNegativos`, `updateInvCounter`, `reportarNegativoDesdeColaborador`).

### 2. Sec. 2 del doc — Inventarios inteligentes
- **A.FISH** abre "Generador de Conteos" → acordeón "Asignar Categorías a Colaboradores" donde marca por colaborador qué `cat` cubre.
- Al pulsar "Generar Inventario Dividido", la función pura `dividirProductos` ([inventario/division.js](../inventario/division.js)) reparte:
  - Productos con categoría asignada → al colaborador dueño.
  - Productos sin dueño → round-robin equitativo (50/3 → 17/17/16 ✓ probado).
- Se crea **una tarea por colaborador** en vez de una sola para "TODOS".

### 3. Sec. 8 — Sistema de negativos
- **Botón "Gestionar Negativos"** en herramientas de A.FISH → modal donde puede:
  - Agregar manualmente (EAN, desc, categoría).
  - Ver lista de negativos reportados por colaboradores.
  - Pulsar "Repartir Negativos" → usa la misma `dividirProductos` → crea tareas `Verificación Negativos`.
- **Botón "+ Reportar Negativo Encontrado"** dentro del modal de ejecución de inventario para que el colaborador agregue negativos a la lista global.

### 4. Sec. 4 — Dashboard ejecutivo F.CEBALLOS
Nav-link exclusivo "Dashboard Ejecutivo" (solo F.CEBALLOS lo ve). Contiene:
- **4 velocímetros SVG semicirculares** ([gerencia/gauge.js](../gerencia/gauge.js)) — sin dependencias, gradiente verde→amarillo→rojo:
  - Productividad (promedio de productividades por líder)
  - Velocidad de ejecución (% completadas dentro de `dueAt`)
  - Confiabilidad inventario (100 - error% basado en `dif`)
  - Cumplimiento operativo (promedio de las 3 anteriores)
- **Tabla "Progreso por Líder"**: asignadas / completadas / pendientes / rendimiento%, con colores semafóricos.
- KPIs son funciones puras en [gerencia/kpis.js](../gerencia/kpis.js): `calcularProductividad`, `calcularVelocidad`, `calcularConfiabilidad`, `calcularCumplimiento`, `statsPorLider`.

### 5. Sec. 5 — Control de merma y averías
- Nueva colección `mermas` ([mermas/data.js](../mermas/data.js)) con **6 entradas seed** para que el dashboard arranque con datos.
- **Flujo automático**: al completar una tarea de inventario, `derivarMermasDeInventario` ([mermas/calculo.js](../mermas/calculo.js)) escanea `task.results.items` y crea una merma por cada `dif < 0` con motivo `'Diferencia Inventario'` y valor `cantidad * PRECIO_PROMEDIO (50)`.
- **Flujo manual**: tool card "Registrar Merma" para F.CEBALLOS → modal con producto/EAN/cantidad/valor/motivo.
- Dashboard muestra: card "Merma del Día", card "Total Histórico", card "Top 5 productos con merma", tabla "Historial Reciente" (últimas 20).

### 6. Sec. 6 (parcial) — Deadlines en tareas
- Modelo de tarea extendido con `createdAt`, `dueAt`, `completedAt`, `assignedBy`.
- Modal "Asignar Nueva Tarea Manual" de líder ahora tiene input `datetime-local` para fecha/hora límite (opcional, default 24h).
- Todos los call sites actualizados: inventario, negativos, líder, ejecutar.
- **Falta**: justificación obligatoria cuando se completa fuera de tiempo, notificaciones de vencimiento.
